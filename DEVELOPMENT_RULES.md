# DEVELOPMENT_RULES.md

このファイルは、Flask（Python製のWebアプリ用フレームワーク）アプリ「いえ電シェア」を実装・修正する際の共通ルールをまとめたものです。

このプロジェクトでは、既存構成を大きく変えず、モック・ER図・既存実装・各チャットで与えられる指示に従って、最小変更で安全に実装を進めることを重視します。

以降のルールは、このプロジェクト内でコード生成・修正・提案を行う際の常設ルールです。
各チャットで追加される今回限りの要件がある場合は、それらも必ず確認したうえで、矛盾しない範囲で両方を守ってください。

特に以下を重視してください。

- 既存コードを壊さない
- モックの見た目を変えない
- 必要最小限の変更だけ行う
- セキュリティと認可を優先する
- 出力は差分中心で行う

以下の内容を前提ルールとして扱ってください。

================
■ アプリ概要
================

アプリ名：
いえ電シェア

家庭内の冷暖房機器の使用時間を記録し、
電気料金を公平にシェアするWebアプリです。

================
■ 最重要ルール
================

各チャットでの添付資料と指示に合わせて、
最小変更・最小追加で、確実に動く実装をしてください。

見た目・HTML構造・class名・id名・文言・レイアウト・配色・余白は、
各チャットで添付したモックをできるだけそのまま使ってください。

私はモックのデザインをかなり作り込んでいるため、
こちらが明示的に要求した場合を除き、以下は禁止です。

- 勝手にデザインを変える
- 配色を変える
- 余白や配置を変える
- サイズ感を変える
- class名を勝手に変える
- id名を勝手に変える
- UI文言を勝手に変える
- Bootstrap などの別デザインに置き換える
- モックにない装飾を追加する
- レイアウトを簡略化する
- HTML構造を不要に組み替える

デザインは原則としてモックから一切変更しないでください。
実装のために必要な最小限の変更だけ許可します。

許可する最小限の変更例：
- form の action / method 追加
- name 属性追加
- href / url_for の追加
- CSRFトークン埋め込み
- flashメッセージのJinja表示
- Jinja2 による条件分岐や値埋め込み
- 一覧表示用のfor文追加
- select 要素への option の埋め込み
- value の差し戻し
- 家アイコンや各ボタンのリンク接続
- 最小限の JavaScript 追加（本当に必要な場合のみ）

CSS は、モックの見た目を壊さない範囲でのみ最小限の追記をしてください。
見た目改善の提案やリデザイン提案は不要です。

■ 出力ルール
================

こちらが明示するまで、タスクの実行はしないでください。

================
■ models.py についての厳守事項
================

models.py はすでに完成済み・マイグレーション適用済みです。
そのため、models.py 全体を再出力したり、既存定義を作り直したりしないでください。

禁止事項：
- models.py 全体の再生成
- 既存カラム名の変更
- 型の変更
- 制約の変更
- nullable の変更
- リレーションの変更
- マイグレーション整合性を壊す変更

まずは models.py を変更せずに実装可能かを優先してください。

もし本当に必要な場合だけ、
最小変更だけを提案してください。

その場合の出力ルール：
- models.py は全文を出さない
- 変更箇所だけ抜粋で示す
- どこに何を追加するか明示する
- 既存定義を壊さないこと

================
■ モックHTML / モック画像の扱い
================

添付したモックをテンプレートに落とし込む際は、
Jinja2（Flaskテンプレートの埋め込み機能）に必要な最小限の変更だけを加えてください。

具体的に許可する変更例：
- {{ url_for(...) }}
- {{ form.csrf_token }} または同等のCSRF埋め込み
- flash メッセージ表示の for 文
- value の差し戻し
- 一覧表示用の for 文
- select の option 埋め込み
- form 化
- 家アイコンや各ボタンのリンク接続
- 必要最小限の条件分岐

それ以外の、見た目に関わる変更は極力しないでください。

================
■ 日本語コメントの方針
================

生成する Python / HTML / JavaScript には、
提出物として残しても不自然ではない、
簡潔で分かりやすい日本語コメントを付けてください。

コメント方針：
- 「この処理は何をしているか」が分かる
- 長すぎない
- 学習メモっぽすぎない
- 実務や提出物で見ても不自然でない
- 特に重要な処理にだけ付ける
- コメントのせいでコードが読みにくくならないようにする

特にコメントを付けてほしい箇所：
- 一般ユーザー限定ルート
- login_required（ログイン必須） + role='user' のチェック
- 新規追加画面の表示処理
- 使用機器一覧の取得
- 記録新規作成処理
- 所有権チェック
- 日時バリデーション
- 未来日時チェック
- 未終了記録の存在チェック
- flash メッセージ表示
- CSRF対策

※変更箇所のみを提示し、ファイル全文は出力しないでください。

必要であれば、
- requirements への追加内容
- インストールコマンド
も示してください

================
■ 注意
================

以下はしないでください。

- Blueprint 化
- フォルダ構成の大改造
- 認証方式の変更
- 既存の login/logout 実装の作り直し
- 既存の一般ユーザートップ画面ロジックの作り直し
- 既存の管理者画面実装の作り直し
- device_usage_logs / devices / users のカラム名変更
- UIデザインの変更
- モックを無視した新規HTMLの作り直し
- models.py の全面再出力
- 既存マイグレーションとズレる提案
- 見た目改善の提案
- デザインのアレンジ
